1명에서 1,000명까지

Stage 9는 첫 1명까지. 거기서 100명, 1,000명까지의 길은 다른 게임. 채널·기술·비용·인력 모두 진화합니다.

"채널 다양화 + 시스템 진화"

§1. 3 마일스톤 — 100·500·1,000명

각 마일스톤마다 시스템·전략·인력이 바뀝니다

100명
PMF 검증 단계
MRR ~100만원
채널: 블로그 SEO + 스레드 + 직접 영업 (1:1)
기술: Stage 6 그대로 충분 (Vercel + Supabase 기본)
CS: 본인 직접 (일 15분)
가격: 9,900원 유지 + 얼리버드
500명
PMF 확장 단계
MRR ~500만원
채널: + 추천 시스템 본격 + 콘텐츠 자동화 (Stage 4 + 7)
기술: + Rate Limit + 캐싱 + 모니터링 강화
CS: 본인 + 매크로 30개 + AI 초안 (n8n)
가격: + 연 구독 79,000원 활성화
1,000명
초기 성장 단계
MRR ~1,000만원
채널: + 첫 유료 광고 (소액 검증) + 파트너십
기술: + DB 인덱스 강화 + read replica 검토
CS: 시간제 1명 (월 30시간)
가격: + 기업/팀 구독 검토 (B2B 첫 시도)
🎯 마일스톤 평균 시간
0 → 100명: 3~6개월 (가장 어려움. PMF 검증)
100 → 500명: 3~6개월 (시스템화·자동화 단계)
500 → 1,000명: 3~9개월 (성장 곡선 + 채널 확장)

"6개월 안에 100명, 1년 안에 500명, 18개월 안에 1,000명"이 1인 SaaS 평균 궤적.

§2. 채널 다각화 — 한 채널 의존 위험

100명까지 한 채널 OK. 1,000명 가려면 3~5개 채널 동시

🌱 Organic — 블로그 SEO
CAC: ~0원 (시간만 투자)
장점: 누적 자산. 1년 후에도 트래픽
단점: 6개월 이상 걸림. 즉시 효과 X
도구: Google Search Console·Ahrefs·Naver SEO
📱 Organic — SNS (스레드·인스타)
CAC: ~5,000원/구독자
장점: 즉시 반응. 빠른 피드백
단점: 매일 만들어야. 알고리즘 의존
도구: Stage 4 4채널 재활용
🤝 Referral — 추천 시스템
CAC: ~3,000원/구독자 (추천인 무료 1개월)
장점: 가장 신뢰 높음. 전환율 2~3배
단점: 100명 미만 시 효과 X (네트워크 효과 필요)
도구: 추천 코드 + Stage 9 SPEC_TRANSPLANT
🤝 Partnership — 다른 뉴스레터/제품
CAC: 가변 (수익 분배)
장점: 신뢰도 높은 사용자 즉시
단점: 협상·관리 부담
도구: 비슷한 타겟의 뉴스레터 1~2개와 cross-promo
💰 Paid — Meta·Google Ads
CAC: 10,000~30,000원/구독자
장점: 즉시 + 정량적 ROI 측정
단점: PMF 없으면 돈 태움. CAC > LTV 위험
도구: Meta Ads Manager (작은 캠페인부터)
📺 Paid — 인플루언서 협찬
CAC: 가변 (5만~50만원/인플루언서)
장점: 한 번 잘 맞으면 100명+
단점: 매칭 어렵고 ROI 예측 불가
도구: 마이크로 인플루언서 (1~10만 팔로워)
🚨 유료 광고 == "PMF 후 100명 넘은 후"
Stage 9의 Won't Do — 100명 전엔 광고 X. 이유: 뉴스레터 안 좋으면 CAC 5만원 태워도 다음 달 다 떠남. 리텐션 70%+ 검증 후 광고로 진입.

§3. 추천 시스템 — 1인 SaaS의 게임 체인저

DB 스키마 — referrals 테이블 SQL
CREATE TABLE referrals ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), referrer_id uuid REFERENCES users(id), -- 추천한 사람 referred_id uuid REFERENCES users(id), -- 추천받은 사람 code text UNIQUE NOT NULL, -- 예: "JUNHO2026" status text DEFAULT 'pending', -- pending → completed reward_given boolean DEFAULT false, created_at timestamptz DEFAULT now(), completed_at timestamptz ); -- 사용자별 추천 코드 자동 생성 (회원가입 시) ALTER TABLE users ADD COLUMN referral_code text UNIQUE;
🎁 인센티브 설계 — Win-Win
추천인 (기존 구독자): 친구가 결제 시 1개월 무료 (다음 결제 한 달 미루기)
피추천인 (신규): 첫 달 50% 할인 (4,950원)

양쪽 다 혜택이 핵심. 한쪽만 혜택이면 효과 절반.

§5. DB 성능 진화 — 단계별 대응

단계증상해결
~1만 row모든 쿼리 즉시 응답아무것도 안 해도 됨
1~10만 row일부 쿼리 100ms+핵심 컬럼 인덱스 (Stage 4 부록)
10~100만 row구독자 조회 1초+Partial index + 복합 인덱스
100만 row+매일 cron이 5분+Read replica + 쿼리 캐싱
1,000만 row+DB 비용 $100+/월Partitioning + 별도 분석 DB
📊 1,000명 SaaS 데이터 규모 추정
users: ~1,500 row (이탈자 포함)
subscriptions: ~2,000 row
newsletters: ~365 row/년 (일 1회 × 1년)
send_logs: ~365,000 row/년 (1,000명 × 365일)

send_logs만 신경쓰면 됨 — 시간 기반 인덱스 (BRIN) 또는 월별 partition.

§7. 가격 정책 진화 — 단계별 plan 추가

0~100명: 월 9,900원 단일 plan + 얼리버드 50% 할인 (처음 100명)
100~500명: + 연 79,000원 (2개월 무료) — 현금 흐름 + 이탈 감소
500~1,000명: + Premium plan (19,900원, 추가 주제·우선 발송) — ARPU 상승
1,000명+: + Team plan (3인 49,000원) — B2B 첫 시도
2,000명+: + Enterprise (협상가) — 큰 회사·기관 대상
📈 ARPU 진화 시나리오
단일 9,900원 → ARPU 9,900원
+ Premium 20% 전환 → ARPU ~12,000원 (21% 상승)
+ Team 5% 전환 → ARPU ~13,500원 (36% 상승)

같은 1,000명이라도 ARPU 올리면 매출 36% 상승. 신규 가입 늘리는 것보다 쉽고 빠름.

PART A
자동화 개념 & 도구 선택
"왜 1인 창업가에게 자동화가 가장 큰 무기인가" — 시간이 자본인 사람의 첫 도구

왜 자동화인가

반복 작업을 제거하면 창의적 작업에만 집중할 수 있습니다 — 1인 창업가의 시간 = 자본

대기업과 1인 창업가의 가장 큰 차이는 "사람을 채용해서 반복 일을 시킬 수 있는가"입니다. 1인 창업가는 그게 안 됩니다. 그래서 "매일 하는 것"을 "한 번 설계하고 자동 반복"으로 바꾸는 능력이 사실상 채용을 대체합니다.

⏱️ "매일 하는 것" → "한 번 설계 + 자동 반복" 전환 효과
매일 1시간 걸리던 작업을 자동화로 옮기면 월 22시간 + 연 264시간이 자유로워집니다. 이 시간을 콘텐츠 제작·고객 상담·신사업 기획에 쓰면 매출이 올라갑니다. 자동화는 "비용 절감"이 아니라 "수익 창출의 시간 확보"입니다.

PART B
n8n 실전 — Docker 설치부터
"택배 상자 안에 든 도구" — Docker 비유로 5분 안에 시작

설치 + 핵심 개념

Docker 한 번 깔아두면 n8n뿐 아니라 미래에 만날 모든 자체 호스팅 도구가 같은 방식

📦 Docker — 택배 상자 비유

Docker는 "프로그램이 필요한 모든 것을 택배 상자 안에 미리 담아 보내주는 도구"입니다. 내 컴퓨터에 직접 설치 안 하고, 상자(컨테이너)만 실행하면 됩니다. 안 쓸 땐 상자 닫으면 깔끔.

n8n 설치 4단계 (Mac 기준, 5분) Docker
# 1. Docker Desktop 다운로드 (docker.com) # Mac/Windows 모두 GUI로 설치. 한 번만. # 2. 터미널에서 n8n 컨테이너 실행 $ docker run -d --name n8n -p 5678:5678 \ -v ~/.n8n:/home/node/.n8n n8nio/n8n # 3. 브라우저에서 접속 $ open http://localhost:5678 # → n8n 첫 화면 (계정 생성 5초) # 4. API 키 발급 (오른쪽 상단 ⚙ → API) # 이 키를 .env.local의 N8N_API_KEY 에 저장
🔑 Claude Code에 n8n MCP 연결
n8n API 키 발급 후 Claude Code에 /mcp add n8n 또는 ~/.claude/settings.json의 mcpServers에 등록. 이후부터 Claude Code 안에서 "n8n으로 ○○ 워크플로우 만들어줘"가 가능해집니다. 이 한 번의 셋업이 Stage 7의 핵심.
📜 CLAUDE.md에 추가할 토큰 최적화 규칙
n8n MCP는 노드 목록을 매 호출마다 가져오기 때문에 토큰을 많이 먹습니다. CLAUDE.md에 추가:

"n8n 워크플로우 자주 쓰는 패턴 5개는 .claude/skills/ 의 SKILL로 저장. 새 워크플로우 만들 때만 MCP 호출. 단순 조회는 SKILL 우선."

뉴스레터 서비스 5가지 워크플로우

Stage 6 라이브 URL과 연결되는 실전 자동화 — 이대로 만들면 됩니다

1
📰 뉴스 수집 자동화 (매일 06:00)
매일 오전 6시 정해진 뉴스 소스에서 기사를 가져와 Claude로 요약하고 newsletters 테이블에 저장. 8시 발송 워크플로우의 재료가 됩니다.
⏰ Cron 매일 06:00 → HTTP 뉴스 API → Claude AI 요약 → Supabase Insert
2
👋 구독자 알림 자동화 (즉시)
새 구독 신청 webhook 수신 → 환영 이메일(Resend) + 카카오 알림 + 텔레그램 운영자 알림 동시 발송. Stage 6에서 만든 텔레그램 봇이 진가를 발휘.
📨 Webhook 구독 신청 → Resend 환영 메일 + 카카오 알림톡 + 텔레그램 운영자
3
📊 주간 리포트 자동화 (월요일 09:00)
매주 월요일 PostHog에서 지난 주 지표 조회 → Claude로 한국어 분석 → 텔레그램으로 운영자에게 전송. 출근하면서 이미 지표 받음.
⏰ Cron 월요일 09:00 → PostHog API → Claude 분석 → 텔레그램 발송
4
🌐 콘텐츠 배포 자동화
뉴스레터 발행 직후 워드프레스 SEO 포스팅 + Canva로 SNS 카드 생성 + 스레드 자동 발행. 1편 → 4채널 (Stage 4 전략 자동화).
📰 뉴스레터 발행 → WP REST API + Canva Magic Resize + Threads API
5
🔁 구독 갱신 관리 (실패 처리)
토스페이먼츠 결제 실패 webhook → 사용자에게 재시도 안내 메일 → 3회 실패 시 자동 구독 정지 + 운영자 알림. 환불 분쟁 예방.
💳 결제 실패 Webhook → 재시도 안내 메일 → IF 3회 실패 → 구독 정지 + 알림
⚡ n8n 워크플로우 설계 프롬프트
n8n으로 [워크플로우 이름] 자동화를 설계해줘. [워크플로우 이름]: 뉴스레터 자동 발송 (매일 오전 7시) 요구사항: 1. 트리거: Cron 매일 07:00 KST 2. 활성 구독자 조회 (Supabase, status='active' AND active_topics 존재) 3. 각 구독자별 다음 작업: - 구독 주제(IT/AI/투자)에 맞는 newsletters 조회 - Resend로 HTML 이메일 발송 (개인화 인사말 포함) - send_logs에 결과 기록 4. 전체 발송 완료 후 텔레그램 운영자에게 요약 (성공 N건 / 실패 M건) 5. 실패 발송은 자동 재시도 1회 후 send_logs status='failed' 각 노드: - 노드 이름 + 타입 (Cron / Supabase / HTTP / Resend / IF / Telegram) - 입력 데이터 → 출력 데이터 - 핵심 설정 (cron 표현식, SQL, 헤더 등) 결과 JSON으로 출력 (n8n에 import 가능한 형식). 주의사항 3가지 (rate limit, 재시도 폭주 방지, 멱등성).

⚖️ MCP vs Skill 최적화

MCP는 강력하지만 매 호출마다 노드 목록을 가져와 토큰을 잡아먹습니다. 자주 쓰는 워크플로우 패턴은 Stage 3에서 배운 SKILL로 분리하면 토큰 50% 이상 절감.

상황MCPSKILL

이미 있는 워크플로우 재사용 × ⭕ SKILL로 호출
워크플로우 수정·디버깅 ⭕ MCP로 진단 ×
매일 도는 패턴 (cron/webhook) × ⭕ SKILL로 자동 트리거
⚡ Claude Code → n8n MCP 연동 프롬프트
Claude Code에서 n8n MCP를 사용해 다음 워크플로우를 만들어줘. 목표: AI 뉴스레터 서비스의 [워크플로우 이름] 연결: Stage 6 Supabase + Resend + 텔레그램 봇 작업 순서: 1. n8n MCP로 현재 워크플로우 목록 조회 (중복 방지) 2. 동일 이름 있으면 사용자에게 "수정 vs 새로 만들기" 물어봄 3. 새 워크플로우 JSON 생성 (트리거 + 노드 + 연결) 4. n8n MCP로 import + 활성화 5. 첫 실행 테스트 (수동 trigger) 6. 결과 확인 후 cron/webhook 활성화 만든 워크플로우는 .claude/skills/ 폴더에 "workflow-[이름].md" 파일로 자동 저장 (다음번에 SKILL로 재사용). 토큰 절약: MCP 호출은 새 작업 시에만, 재사용은 SKILL 우선.
PART D
카카오톡 자동화
Claude Cowork + PlayMCP 조합 — 비즈니스 계정 없이 카카오 연동

PlayMCP 카카오 자동화

개인 카카오톡으로 본인·베타 테스터 그룹에 알림 — 정식 비즈니스 채널 전 단계

Stage 6에서 봤듯 카카오 정식 알림톡은 비즈니스 채널 등록 + 템플릿 심사(1~2주)가 필요합니다. 그 전까지는 PlayMCP의 "카카오톡 나채방 MCP"를 임시 활용할 수 있습니다. 본인이나 신뢰하는 테스터 그룹에 자동화 알림 가능.

🤖 Claude Cowork + PlayMCP MCP 추가
Claude Cowork는 컴퓨터 조작 에이전트(Stage 1 도구 지도). 여기에 PlayMCP의 카카오톡 나채방 MCP + 네이버 검색 MCP를 추가하면:

흐름: 매일 아침 → 네이버 검색 MCP로 키워드 뉴스 수집 → Claude로 요약 → 카카오톡 나채방 MCP로 본인 채팅방 발송

컴퓨터 켜놓으면 매일 자동. 정식 알림톡 전 MVP 검증용으로 최적.
⏰ D-100 카운트다운 자동 알림 — 응용 사례
"런칭 D-100" 같은 카운트다운을 매일 아침 카카오로 받기:
n8n cron 매일 08:00 → Claude로 "D-N 메시지 + 오늘 할 일 3가지" 생성 → PlayMCP 카카오톡 나채방 발송

매일 같은 시간 받으면 동기부여 + 진척 추적. 1인 창업가의 자기 관리 도구로 강력.
⚠️ 정식 채널로의 전환 시점
유료 구독자가 50명 넘기 전에 카카오 비즈니스 채널 등록 시작. PlayMCP 방식은 본인용·베타용에만 사용. 상용 서비스 알림은 반드시 정식 알림톡으로. 약관·신뢰성 문제.

💡 시간당 가치를 어떻게 정하나
가장 쉬운 방법: 본인이 같은 시간에 만들 수 있는 매출. 콘텐츠 1편이 월 100,000원 가치라면 콘텐츠 만드는 시간이 시간당 가치.
보수적 기준: 본인이 받고 싶은 시급 (예: 30,000원).
공격적 기준: 그 시간에 안 했던 영업·기획 활동의 가치 (예: 100,000원).
⚡ 자동화 ROI 계산 프롬프트
내 비즈니스에서 자동화할 후보 5개의 ROI를 계산해줘. 비즈니스: AI 뉴스레터 자동화 서비스 1인 창업가 시간당 가치: 30,000원 자동화 후보 5개: 1. 뉴스 수집 + AI 요약 (현재 매일 2시간) 2. 신규 구독자 환영 메일 (현재 1회당 5분, 일 평균 4명) 3. 주간 지표 리포트 (현재 매주 1.5시간) 4. SNS 콘텐츠 변환 (현재 1편당 30분, 주 5편) 5. 고객 문의 응답 (현재 1건당 10분, 일 평균 3건) 각 항목별: 1. 현재 소요 시간 (월간 환산) 2. 자동화 후 예상 시간 3. 절감 시간 × 시간당 가치 = 월간 절감액 4. 자동화 구축 시간 (한 번만 드는 비용) 5. 손익분기점 (몇 개월 만에 회수되나) 6. 실행 우선순위 (1~5위) 표 형식으로 정리. 마지막에 "1번부터 시작"의 근거.

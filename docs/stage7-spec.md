## 목차 섹션 (순서대로)

### 파트 A: 자동화 개념 & 도구 선택

자동화가 왜 핵심인가:
- 반복 작업을 제거하면 창의적 작업에 집중 가능
- 1인 창업가의 시간 = 자본
- "매일 하는 것" vs "한 번 설계하고 자동 반복"

도구 3가지 비교 (표):
- Zapier: 쉬움, 비쌈, 단순 연결
- Make.com: 중간 난이도, 시각적 인터페이스, SaaS 연동
- n8n: 자체 호스팅 무료, Claude Code MCP 연동 최강

언제 뭘 쓰나:
- 지금 당장 빠르게 → Make.com
- 복잡한 비즈니스 로직 → n8n
- n8n vs Make 비용 판단 기준 (언제 n8n이 200% 이익인가)

### 파트 B: n8n 실전 (Docker 설치부터)

n8n 설치 (n8n + Claude Code 파일 활용):
- Docker Desktop 설치 (택배 상자 비유)
- n8n 컨테이너 실행 (localhost:5678)
- n8n API 키 발급
- Claude Code에 n8n MCP 연결

CLAUDE.md에 추가할 토큰 최적화 규칙

n8n 핵심 개념:
- 워크플로우 = 자동화 전체 캔버스
- 노드 = 개별 작업 단위
- 트리거 노드 = 시작점 (시간/웹훅/이벤트)
- 연결 = 데이터 흐름

실전 워크플로우 5가지 (뉴스레터 서비스 기준):

1. 뉴스 수집 자동화
   트리거(매일 오전 6시) → HTTP 뉴스 API → Claude AI 요약 → DB 저장

2. 구독자 알림 자동화
   구독 신청 Webhook → 환영 이메일(Resend) → 카카오 알림 → 텔레그램 관리자 알림

3. 주간 리포트 자동화
   트리거(매주 월요일) → PostHog 지표 조회 → Claude 분석 → 텔레그램 전송

4. 콘텐츠 배포 자동화
   뉴스레터 발행 → 워드프레스 포스팅 → SNS 카드 생성(Canva) → 스레드 발행

5. 구독 갱신 관리
   결제 실패 Webhook → 재시도 알림 이메일 → 3회 실패 시 구독 정지

### 파트 C: Claude Code + n8n MCP 통합
(n8n + Claude Code 파일 내용 활용)

연동 구조:
Claude Code ↔ n8n MCP ↔ n8n 워크플로우

Claude Code에서 워크플로우 만들기:
"n8n MCP를 활용해서 [워크플로우 설명] 만들어줘"

MCP vs Skill 최적화:
- MCP만 쓰면 컨텍스트 낭비 → Skill로 대체
- 자주 쓰는 워크플로우는 Skill로 저장

### 파트 D: 카카오톡 자동화
(카카오 자동화 파일 내용 활용)

Claude Cowork + PlayMCP 조합:
- 카카오톡 나채방 MCP 추가
- 네이버 검색 MCP 추가
- 매일 아침 뉴스 수집 → 분석 → 카카오로 자동 발송

D-100 카운트다운 자동 알림:
컴퓨터 켜놓으면 매일 아침 오늘 할 일 카카오로 전송

### 파트 E: Make.com 실전 입문
- Make.com 무료 계정 시작
- 시나리오 = n8n의 워크플로우
- 구글 시트 → Slack/텔레그램 알림 (가장 기본)
- 뉴스레터 구독자 구글 시트 → 자동 이메일 발송
- n8n과 Make 연동 (상황별 최적 조합)

### 파트 F: 1인 창업가 자동화 TOP 10
비즈니스별 ROI가 높은 자동화:
1. 뉴스 수집 + AI 요약 → 뉴스레터 초안
2. 신규 구독자 → 환영 시퀀스 (3통 자동 이메일)
3. 결제 완료 → 콘텐츠 접근 권한 자동 부여
4. 블로그 발행 → SNS 자동 포스팅
5. 주간 지표 → 텔레그램 자동 리포트
6. 유튜브 업로드 → 자막 추출 → 블로그 변환
7. 고객 문의 → AI 초안 → 검토 후 발송
8. 구독 해지 → 재가입 유도 시퀀스
9. 생일 구독자 → 특별 혜택 자동 발송
10. 월말 정산 → 수익 리포트 자동 생성

자동화 ROI 계산기:
절감 시간(시간/일) × 22일 × 시간당 가치 = 월간 절감액

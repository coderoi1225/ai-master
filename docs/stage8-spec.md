핵심 MCP 5개 실전 연결:

MCP 연결 후 할 수 있는 것 (뉴스레터 서비스):

### 섹션 2: Subagents & HITL

Subagent란 (자비스 라우터 개념 심화):
- 메인 에이전트 = 오케스트레이터 (자비스)
- 서브에이전트 = 전문 팀원 (작업별 담당)
- 서브에이전트 간 직접 통신 금지 → 메인 통해 조율

뉴스레터 서비스 멀티에이전트 구조:
- 오케스트레이터: 전체 워크플로우 감독
- 리서처: 뉴스 수집 + 요약
- 라이터: 뉴스레터 초안 작성
- 리뷰어: 품질 검토 + 팩트체크
- 퍼블리셔: 발송 실행

HITL (Human-in-the-Loop):
- 언제 사람이 개입해야 하나
- 자동 vs 승인 필요 결정 기준
- 뉴스레터 발송 전 사람 검토 단계 설계

### 섹션 3: RAG + Dify 연동

RAG (Retrieval-Augmented Generation) 개념:
- 일반 AI vs RAG AI 차이
- 내 문서로 답하는 AI (살아있는 지식베이스)
- 벡터 데이터베이스 개념 (의미 기반 검색)

Dify 실전:
- Dify 가입 + 지식베이스 생성
- 문서 업로드 (PDF, Markdown, URL)
- 지식베이스 → AI 앱 연결
- API 연동: 현재 AI 튜터 fetch URL 교체

뉴스레터 서비스 Dify 활용:
- 지식베이스: 발행한 뉴스레터 전체 아카이브
- 구독자가 "지난주 AI 트렌드 뭐였어?"라고 물으면
- Dify가 아카이브에서 검색해서 정확하게 답변

VIBE MASTER와 Dify 연결:
- 각 Stage HTML AI 튜터 → Dify RAG로 업그레이드
- fetch URL을 Dify API로 교체하는 방법

### 섹션 4: 에이전트 거버넌스

왜 필요한가:
- 에이전트가 실수할 때 (rm -rf 같은)
- 권한 범위를 어떻게 설정하는가
- 감사 로그: 에이전트가 한 일 추적

실전 거버넌스 설계:
- settings.json으로 위험 명령 차단 (복습 + 심화)
- 에이전트 권한 범위 정의 (읽기 전용 vs 쓰기)
- 실패 복구 전략 (Harness error → pending 재시도)
- Checkpoint + Rewind 활용

뉴스레터 서비스 거버넌스:
- 발송 에이전트: 구독자 DB 읽기 가능, 삭제 불가
- 결제 에이전트: 조회만 가능, 환불은 사람 승인
- 콘텐츠 에이전트: 초안 작성만, 발송은 사람 확인

### 섹션 5: 컨텍스트 엔지니어링 심화

메타 프롬프팅:
- 프롬프트가 프롬프트를 만드는 구조
- 자동 자기검증 루프 (autoresearch 스킬 활용)
- 재귀적 프롬프트 체인

장PM 메타 스킬 심화 활용:
- blueprint → deep-dive → 구현 → autoresearch → reflect
- 뉴스레터 서비스 전체 워크플로우에 적용

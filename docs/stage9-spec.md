## 목차 섹션 (순서대로)

### 섹션 1: AARRR — 성장 지표 프레임워크

AARRR이 왜 마지막이 아닌가:
- Stage 1에서 NSM 정함 → Stage 6에서 PostHog 심음 → Stage 7에서 자동 리포트 → 여기서 해석
- 지표는 쌓여있음. 이제 읽는 법을 배움

5단계별 뉴스레터 서비스 기준:
- Acquisition(획득): 어디서 구독자가 오는가 (블로그/SNS/추천)
- Activation(활성화): 첫 뉴스레터를 열어봤는가
- Retention(유지): 3개월 후에도 구독 중인가
- Revenue(수익): 유료 전환율, ARPU, LTV
- Referral(추천): 추천 코드로 친구 데려오는가

AARRR 깔때기 분석:
- 방문자 1,000명 → 구독 신청 100명 → 유료 전환 10명
- 어디서 막히는가 (병목 찾기)
- Claude에게 시키는 법: "PostHog 데이터 분석해서 병목 알려줘"

### 섹션 2: NSM 측정 & A/B 테스트

NSM 복습 (Stage 1에서 정한 것):
- 월간 활성 유료 구독자 수
- 이 숫자를 올리기 위한 실험 설계

A/B 테스트 실전:
- PostHog Feature Flags로 설정
- 뉴스레터 제목 A vs B 실험
- 발송 시간 오전 7시 vs 오전 8시 실험
- 결과 해석: 통계적 유의성 판단

PMF (Product-Market Fit) 측정:
- Sean Ellis Test: "이 서비스가 없어지면 얼마나 실망하겠어요?" 40% 이상 = PMF
- 4가지 PMF 시나리오:
  1. PMF 달성 → 확장 (광고·채널 투자)
  2. 고객층 문제 → 페르소나 재설정
  3. 콘텐츠 문제 → 주제·형식 변경
  4. 채널 문제 → 유입 경로 바꾸기

### 섹션 3: KPT 회고 & 6개월 로드맵

KPT 회고 (30분 의식):
- Keep: 잘 작동한 것
- Problem: 문제였던 것
- Try: 다음에 시도할 것
- Claude 대화형 회고 프롬프트 (8개 질문)

6개월 로드맵 작성:
- 분기별 OKR (목표 1개 + 핵심 결과 3개)
- 매주·매달 측정 지표
- Won't Do 리스트 (안 할 것 명시)
- 재진단 트리거 (어떤 신호가 오면 방향 바꾸나)
- 파일명: docs/Roadmap-2026H2.md

### 섹션 4: 성공 앱 리버스엔지니어링 & 이식

Phase 1 — Antigravity로 성공 앱 해체:
- 타겟 앱 선정 (뉴스레터 분야 성공 사례)
- 바이럴 루프 분석 프롬프트
- "광고 없이 1만 명을 모은 핵심 기능 3가지"
- 심리적·기술적 트리거 분석

Phase 2 — SPEC_TRANSPLANT.md 작성:
- 성공 앱의 핵심 기능을 내 서비스 맥락으로 재설계
- 기존 DB 스키마와 충돌 없이 이식
- Claude Code로 SDD (Spec-Driven Development) 구현

Phase 3 — 무광고 성장 엔진 장착:
- SEO 자동화: 동적 메타태그 컴포넌트
- 바이럴 루프: 추천 코드 시스템
- 커뮤니티 전략: 오픈 카카오 채널

### 섹션 5: 첫 판매 전략

0단계 — 수동 검증:
- 지인 5명에게 URL 보여주기
- "얼마면 살 거예요?" 물어보기
- 결제 전 수작업으로 뉴스레터 발송해보기

단계별 판매 계획:
1. 소프트 런치: URL 공유 → 첫 유료 구독자
2. 첫 10명: 직접 영업 + 개인화 커뮤니케이션
3. 50명: 블로그 SEO + SNS
4. 100명: 추천 프로그램 활성화

가격 전략:
- 월 9,900원 vs 연 79,000원 (2개월 무료)
- 얼리버드 50% 할인 (처음 100명)
- 무료 체험 14일

### 섹션 6: 안 할 것 리스트 (Won't Do)
